# Contributing to RedisVL TypeScript

Thanks for your interest in contributing. The full contributor guide lives on the documentation site:

**https://redis-developer.github.io/redis-vl-typescript/docs/contributing**

The source for that guide lives in [`website/docs/contributing/`](./website/docs/contributing/) and is structured as:

- [Getting Started](./website/docs/contributing/getting-started.md) - prerequisites, setup, development workflow, reference implementations
- [Testing & Validation](./website/docs/contributing/testing.md) - running tests, validation pipeline, recipe smoke test, CI
- [Code Standards](./website/docs/contributing/code-standards.md) - type discipline, naming, file organization, cross-language API parity
- [Git Workflow](./website/docs/contributing/git-workflow.md) - branch naming, conventional-commit format
- [Documentation Guidelines](./website/docs/contributing/documentation.md) - what to document, what not to document

## Quick start

```bash
git clone https://github.com/redis-developer/redis-vl-typescript.git
cd redis-vl-typescript
npm install
npm test
```

For anything beyond that - TDD workflow, validation gates, commit format - read the contributor guide above.

## Reporting bugs and proposing features

Open an issue at https://github.com/redis-developer/redis-vl-typescript/issues.
